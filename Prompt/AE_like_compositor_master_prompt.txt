# 1) Цель продукта
Спроектировать и реализовать open-source кроссплатформенный motion graphics/VFX-композитор для:
- Linux, Windows, macOS (desktop first),
- с архитектурой, пригодной для будущего mobile/viewer,
- с UX, знакомым пользователям AE (таймлайн, слои, keyframes, graph editor, precomp),
- с современным realtime playback и ускоренным рендером (GPU/CPU).

Важно:
- Называй продукт “AE-like compositor”, “motion compositor”, “open compositor”.
- Не предлагай нарушающие лицензии действия.
- Формулируй совместимость через “импорт/экспорт открытых и документированных форматов”.

# 3) Продуктовые требования (MVP -> v2)
## MVP (обязательное)
- Проекты и сцены:
  - project file (версионируемый формат, например JSON/MsgPack + schema version),
  - compositions, layers, nesting (precomp),
  - assets management (video/image/audio/solid/text).
- Таймлайн:
  - trim/split, in/out, snapping, markers,
  - keyframes (linear, bezier, hold),
  - graph editor (value/speed).
- Трансформации и базовые эффекты:
  - position/scale/rotation/opacity/anchor,
  - blend modes (основные),
  - masks (bezier), feather, track matte,
  - базовый color correction (levels/curves/exposure/saturation),
  - blur, glow, transform, fill/tint.
- Текст:
  - text layers, basic typography, animators (минимум opacity/position/range).
- Превью/рендер:
  - RAM preview/realtime playback,
  - background render queue,
  - export: image sequence, ProRes/H.264 (через FFmpeg), alpha support где возможно.
- Аудио:
  - waveform preview, basic sync, volume keyframes.
- UX:
  - workspace/panels, hotkeys, undo/redo, autosave, crash recovery.
- Платформы:
  - Linux (первый класс), Windows, macOS.

## v2 (после MVP)
- 3D layers (без full 3D сцены на старте),
- expressions (безопасный sandbox-язык),
- motion blur, parenting, nulls, shape layers advanced,
- plugin SDK,
- collaborative/cloud pipeline (опционально).

# 4) Техническая архитектура (предпочтительный стек)
Предложи и аргументируй один основной стек и один резервный.

Кандидат A (рекомендуемый):
- Core/Engine: Rust
- UI: Qt (C++/QML) или Tauri + Rust + custom panels (обосновать выбор)
- Render graph: wgpu (Vulkan/Metal/DX12 backend)
- Media I/O: FFmpeg
- Audio: CPAL/rodio или PortAudio
- Text shaping: HarfBuzz + FreeType
- Color: OCIO (позже), в MVP — linear/sRGB pipeline
- Scripting: Lua/WASM sandbox (v2)

Кандидат B:
- Core: C++20
- UI: Qt
- Render: Vulkan + abstraction layer

Требования к архитектуре:
- модульность (engine, timeline, effects, io, ui, render backend),
- deterministic render path,
- data-oriented подход для playback,
- cache system (disk + RAM),
- job system/thread pool,
- capability detection по GPU/драйверам,
- graceful fallback на CPU.

# 5) Ключевые инженерные подсистемы
Проработай подробно:
1. Project model + serialization + migration strategy
2. Timeline engine + keyframe interpolation + subframe timing
3. Render graph + invalidation + dependency tracking
4. Effect pipeline (node-based internals, layer-based UX)
5. Caching strategy (frame cache, tile cache, proxy workflow)
6. Playback architecture (real-time constraints, frame skipping policy)
7. Color pipeline (linear workflow, LUT support roadmap)
8. Media ingest/transcode/proxy generation
9. Crash safety, autosave, telemetry (opt-in), diagnostics
10. Plugin API (ABI/versioning/sandbox/security model)

# 6) Совместимость и форматы
- Сразу поддержать:
  - import: PNG/JPEG/EXR, WAV/MP3, MOV/MP4 (через FFmpeg),
  - export: image sequence + common video codecs.
- Для “совместимости с AE-пайплайном”:
  - импорт/экспорт через промежуточные форматы (JSON, OpenTimelineIO при возможности),
  - не обещать нативную 100% совместимость .aep на старте.
- Дай матрицу совместимости по релизам.

# 7) UI/UX требования
- Layout как у pro tools: Project, Timeline, Composition Viewer, Effects, Graph Editor.
- Сильные hotkeys и customization.
- Linux-first стабильность: Wayland/X11, HiDPI, multi-monitor.
- Accessibility: scalable UI, high-contrast theme, keyboard-first навигация.
- UX-принципы:
  - минимальная latency действий,
  - предсказуемость keyframe/edit поведения,
  - “pro defaults”, но onboarding presets для новичков.
